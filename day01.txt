Typescript


타입스크립트를 사용하는 이유는 안정성 때문이다.
타입 안정성 덕분에 타입스크립트의 장점을 이용할 수 있다.
= 개발자 경험을 누릴 수 있음
= 코드에 버그가 엄청 많이 줄어들게 된다
= 런타임 에러가 줄어든다
= 생산성이 늘어난다 

---------------------------------------------------------------

타입스크립트는 자바스크립트 코드로 컴파일 된다.
타입스크립트는 코드가 실행되기 전에 에러를 확인해준다.
코드에 에러가 있으면 그 코드는 자바스크립트로 컴파일되지 않는다.

타입스크립트가 성공적으로 컴파일 되어서 자바스크립트 코드를 받으면,
타입스크립트 코드가 제대로 작성되었고, 데이터타입에도 문제가 없었다.
=> 자바스크립트 코드에 버그가 전혀 없었다는 뜻

⭐ Type 시스템
└ 명시적 정의(변수 선언 시 타입 정의)
let a: boolean = "x"
→ 🚫 boolean 타입에 string타입 할당 불가 알림
└ 변수만 생성(타입 추론)
let b = "hello"
→ b가 string 타입이라고 추론
b = 1
→ 🚫 string 타입에 number타입 할당 불가 알림
